Данный проект представляет собой полноценное веб-приложение для создания и прохождения опросов с поддержкой аутентификации, однократного голосования и отображения статистики в реальном времени. Проект состоит из двух независимых, но взаимосвязанных частей: бэкенда на Django (REST API + PostgreSQL) и фронтенда на React (SPA с интерфейсом в стиле Ведьмака). Архитектура следует принципам разделения ответственности: бэкенд отвечает за хранение данных, бизнес-логику и безопасность, а фронтенд — за пользовательский интерфейс и взаимодействие.

Бэкенд (/backend) реализован с использованием Django 6.0 и Django REST Framework. Вся логика разделена на три основных приложения:

    polls — содержит модели данных:
        Poll (опрос),
        Question (вопрос),
        Choice (вариант ответа),
        Vote (голос пользователя).

    Модель Vote имеет уникальное ограничение unique_together = ('user', 'choice'), что гарантирует возможность голосования только один раз за вариант.
    
    api — предоставляет RESTful API через сериализаторы и представления:
        ChoiceWithStatsSerializer вычисляет vote_count, percentage и my_votes (определяет, голосовал ли текущий пользователь),
        PollList отдаёт список опросов гостям и авторизованным,
        VoteCreate обрабатывает голосование с проверкой уникальности,
        эндпоинты /api/login/ и /api/register/ реализуют аутентификацию через токены (TokenAuthentication).
    
    polling_project — главный конфигурационный модуль Django с настройками подключения к PostgreSQL, CORS (для взаимодействия с React), аутентификации и безопасности.

Фронтенд (/frontend) создан с помощью React и Bootstrap, оформлен в тематике Ведьмака. Основной компонент App.js реализует полную логику взаимодействия:

    Гость видит опросы, но не может выбирать варианты.
    Авторизованный пользователь может выбрать по одному варианту на вопрос, после чего нажимает «Отправить наше мнение».
    После отправки выбор фиксирован, отображается статистика (N голосов, X%) и пометка «✓ Ваш выбор».
    Имя пользователя (логин) отображается над кнопкой «Выйти».
    Все состояния (авторизация, выбор, отправка) управляются через useState и useEffect, а данные сохраняются в localStorage для сохранения сессии между перезагрузками.
